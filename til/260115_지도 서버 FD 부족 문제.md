### 문제 상황
- 서버 재부팅하여 지도 서버도 재부팅해야 하는 상황. 그러나 재부팅이 안 됨
- 재부팅 명령어 실행했을 때 오류 내용은 아래와 같음
- GPT는 파일 디스크립터(FD) + inotify watch가 누적되어 임계치를 넘었기 때문에 발생한 오류라고 함.
- 따라서 system-wide open files 제한, inotify watch 한계 늘려 놓으려고 했었음

```bash
root@web-server:~/web-backend# systemctl start maptiler
Failed to allocate directory watch: Too many open files
```

---

### 해결 방법
- 일시적으로 maptiler + systemd + docker가 필요한 파일 디스크립터(FD)를 기존 한계(1024/4096) 때문에 못 얻고 있었고, ulimit -n 65535로 그 순간 FD가 열리면서 서비스가 정상 기동됨

```bash
ulimit -n 65535
```

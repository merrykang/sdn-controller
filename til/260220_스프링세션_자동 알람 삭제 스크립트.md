### 스프링 세션
- 스프링 세션은 명시적으로 로그아웃하지 않으면 세션이 서버에 남아 있음
- 스프링 세션의 원칙
  - 서버 메모리 / Redis / JDBC 등에 세션 저장
  - 브라우저 창 닫는다고 세션이 바로 삭제되지 않음
  - **세션 타임아웃 전까지 유지**
- 관련 코드
```java
public boolean isUserSessionExists(String userId) {
    if ("master@sbit.co.kr".equals(userId))
        return false;
    Map<String, ? extends Session> sessions = sessionRepository.findByPrincipalName(userId);
    return !sessions.isEmpty();
}
```
###### 스프링 세션 > 기타 정보
1. Spring Session의 진짜 목적
- 애플리케이션 서버와 세션 저장소를 분리
- 기존: Tomcat 메모리 세션 <-> Spring Session: Redis/JDBC로 외부 저장
- 이를 통해 무중단 배포 / 다중 WAS / 수평 확장할 수 있게 함

2. 동작 구조
Browser
   ↓ (JSESSIONID)
Spring Boot
   ↓
Spring Session
   ↓
Redis / DB

3. 저장 구조(redis 기준)
```text
spring:session:sessions:<sessionId>
spring:session:index:org.springframework.session.FindByIndexNameSessionRepository.PRINCIPAL_NAME_INDEX_NAME:<username>
```
- findByPrincipalName()은 이 인덱스 키를 조회하는 것

4. 동시 로그인 제어는 Spring Session이 아니라 Spring Security 영역

```bash
while true
do
  deleted=$(mysql -N -B -uroot -p'1q2w3e!Q@W#E' sdn -e "
  DELETE FROM ui_alarm
  WHERE log_timestamp >= NOW() - INTERVAL 200 DAY
  AND log_timestamp < NOW() - INTERVAL 5 DAY
  AND (
      message LIKE '%정상적인 환경입니다%' OR
      message LIKE '%스위치 인증이 해제된 상태입니다%' OR
      message LIKE '%스위치와 연결이 해제되었습니다%' OR
      message LIKE '%ssh 연결 거절%' OR
      message LIKE '%스위치 SSH 연결 시간(60초) 초과%' OR
      message LIKE '%인증된 장비입니다%'
  )
  LIMIT 1000;
  SELECT ROW_COUNT();
  ")

  echo "Deleted rows: $deleted"

  end_time=$(date +%s)
  elapsed=$((end_time - start_time))

  total_deleted=$((total_deleted + deleted))

  echo "[$(date)] Batch: $batch | Deleted: $deleted | Elapsed: ${elapsed}s | Total: $total_deleted"

  if [ "$deleted" -eq 0 ]; then
    echo "No more rows to delete. Exit."
    break
  fi

  sleep 5
done
```
